# 罫線アナライザー 引継書

別チャットへ引き継ぐための現状まとめ。新しいチャットでは、まずこのファイルを読んでもらえばOK。

## 1. これは何
- 日本株（日経225）のテクニカル分析Webアプリ（罫線アナライザー）。
- さらに「毎日クラウドで自動スクリーニング → サイン銘柄をLINEへレポート通知」する仕組みを構築済み。
- 個人用ツール。判定の予測力はバックテストで弱め（候補出し程度に使う前提）。

## 2. いまの稼働状況（重要）
- **アプリ**: Vite + React。中心は `src/App.jsx`（1ファイルに集約）。**recharts は v2 必須**（v3だと即クラッシュ）。
- **GitHub**: https://github.com/isa198-wq/keisen-analyzer （**Public**、`main` にpush済み）。
- **クラウド自動実行**: GitHub Actions（`.github/workflows/daily-screening.yml`）。**毎日 07:30 JST** 実行 + 手動実行可。テスト実行=**全green（成功）**。yfinanceのクラウドブロックは今のところ無し。
- **通知経路**: GitHub Secret `WEBHOOK_URL`（= Make.com の Webhook URL）→ Make のシナリオ（Custom webhook → LINE「Send a Broadcast Message」）→ **LINE**。Make側で payload の `content` をLINE本文にマッピング済み。**レポート形式で配信中・動作確認OK**。
  - ⚠️ Webhook URL は**秘密情報**。Secret内のみに存在。コード/チャットに**出さない・コミットしない**。
- **ローカル実行**（Windows、任意。クラウドに移行済みなので必須ではない）: `毎日スクリーニング.bat` / `タスク登録.bat` / `タスク削除.bat`。

## 4. 仕様メモ・ハマりどころ
- recharts **v2必須**（v3でクラッシュ）。
- データは **yfinance**（Stooqはbotブロックで不可）。日本株コードは `.T` 付与。
- 縦持ちCSVの日付判定は**区切り文字必須**（`-`/`/`）。でないと "1332" 等4桁コードが「西暦1332年」と誤判定される。
- 多銘柄解析は**チャンク非同期**（同期だとUIフリーズ）。詳細は memory `bulk-screening-architecture` 参照。
- 通知の判定は **App.jsxと完全一致**（生成方式で担保。検証済み: 強い買い41/強い売り39が両者一致）。
- **LINE Notifyは終了** → Make.com経由でLINE。LINEの「Reply Message」はreplyToken必須で定時pushには使えない → **Broadcast/Push** を使う。
- バックテスト（過去約6か月）: 買い/売りの予測エッジは弱い。
