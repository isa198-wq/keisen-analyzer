# イナゴ・トレード盤 バックテスト監査 移植レポート

> **決定（2026-07-06・ユーザー確定）**: 本監査の結果（5回目の採用ゼロ）を受け、**価格・出来高のみのシグナル研究はクローズ**。イナゴ盤は「シグナル生成器」ではなく**状況把握ダッシュボード**（テーマ過熱度・地合いの可視化）として位置づける。CFG閾値の再チューニングは行わない。エッジ探索のリソースはv5（開示情報レイヤー）に集中。将来イナゴを再検証する場合は「小型株ユニバース＋情報レイヤー込み」の形に限る。

対象: `inago_offline.html`（実際に使われているオフライン版）
参照実装: `inago_daily_FIXED.html`（Fable監査による修正済みdaily版、P-1〜P-9を `AUDIT-FIX` コメントで特定）
作業日: 2026-07-05〜07-06

## 1. 現状確認（Step 1）

- `inago_offline.html` は外部ライブラリ非依存の自前SVG/JS版。`scoreAt(candles,i)` でバー状態判定→`fwd=(c[i+N].close/base-1)*100` の集計構造はdaily版と同一。
- data.js は既存プロジェクト内にもDownloadsにも存在せず、新規生成が必要だった。

## 2. data.js生成（Step 2）

ユーザー指示により、ウォッチ銘柄ではなく **日経225全銘柄（223銘柄・5年）** を母集団に採用。既存の `screening_data.csv`（`fetch_data.py`／yfinance／`auto_adjust=True`）を流用し [build_data_js.mjs](build_data_js.mjs) で `window.INAGO_DATA` 形式に変換。

**変換時に発見・修正したバグ**: CSVがCRLF改行のため `.split("\n")` だけでは各行末列（出来高）に `\r` が混入し、全銘柄の出来高が `NaN` 化していた。`split(/\r?\n/)` に修正し再生成（[data.js](data.js)、223銘柄、asOf=2026-07-03）。この状態のまま検証していたら「初動候補が一度も出現しない」という誤った結論になるところだった。

## 3. P-1〜P-9の移植（Step 3）

全項目がoffline版に適用対象として存在した（「対象外」項目なし）。[inago_offline.html](inago_offline.html) に `AUDIT-FIX #P-n` コメント付きで移植済み。バックアップ: [inago_offline_before.html](inago_offline_before.html)。

### P-9の実証
旧ロジックの `regimeAtOffset()` は中立時に文字列 `"neutral"` を返すが、集計オブジェクトのキーは `"neu"`。ブラウザで実データを読み込み `検証` タブをクリックしたところ、`acc["neutral"]` が `undefined` となり `rb.n++` で例外→パネルが白紙のまま停止することを確認。修正版では地合い判定不能日は `null` を返し `if(reg){...}` でスキップするため再現せず、テーブルが正常表示されることを確認済み。

## 4. 新旧比較（Step 4・223銘柄・実データ）

状態別「件数／勝率／期待値／平均MFE／平均MAE」（期待値=平均、旧仕様: 当日終値・エピソード非圧縮・コスト0・ギャップ除外なし／新仕様: 翌寄付・エピソード圧縮・コスト0.2%・ギャップ8%除外）:

**初動候補**
| N | 旧: 件数/勝率/期待値 | 新: 件数/勝率/期待値 |
|---|---|---|
| 3日 | 4025 / 51.2% / +0.128% | 3765 / 47.8% / **-0.074%** |
| 5日 | 4020 / 52.0% / +0.377% | 3760 / 50.1% / +0.174% |
| 10日 | 4018 / 55.4% / +1.026% | 3758 / 53.1% / +0.798% |

参考: 継続・過熱・出口・様子見も同様に劣化するが、初動候補との相対順位（初動候補 > 過熱・出口の期待値）は新仕様でも概ね維持（N=10のみ過熱が初動候補をわずかに上回る）。

### 帰属分析（初動候補・期待値シフト、OLD基準から1修正ずつON）
| 修正 | N=3 | N=5 | N=10 |
|---|---|---|---|
| P-1 翌寄化のみ | -0.019 | -0.018 | -0.017 |
| P-3 エピソード圧縮のみ | **+0.031** | +0.032 | +0.012 |
| P-4 コスト0.2%のみ | **-0.200** | -0.200 | -0.200 |
| P-6 ギャップ8%除外のみ | -0.018 | -0.020 | -0.023 |

**コスト控除（P-4）が劣化のほぼ全て**を占める（全Nで-0.2%固定、コストがそのまま乗るだけなので当然）。翌寄化・ギャップ除外は小幅なマイナス。エピソード圧縮は逆に軽微なプラス（連続シグナルの重複がむしろ弱い足を水増ししていた）。

### ツール自身の「機能している」判定
`works` 判定（初動候補n≥20 かつ 期待値>0 かつ 期待値>過熱 かつ 期待値>出口）は、**新仕様では N=3/5/10 の全てでFalse**（N=3は期待値自体がマイナス、N=5/10は出口・過熱が初動候補を上回るため）。旧仕様の緩い基準では成立して見えていた可能性がある。

### 追加所見: 地合い別内訳が想定と逆
ツールの `REGIME_VERDICT` は「リスクオンで初動を取りに行ける」という設計だが、実データ(N=5・新仕様)では:

| 地合い | n | 勝率 | 平均 | 中央値 | t値 |
|---|---|---|---|---|---|
| リスクオン | 2491 | 48.4% | +0.048% | -0.140% | 0.52（有意でない） |
| リスクオフ | 590 | 53.2% | +0.518% | +0.281% | 2.02（境界的に有意） |
| 中立 | 679 | 53.5% | +0.337% | +0.338% | 1.75（有意でない） |

リスクオンの中央値はマイナス（平均は少数の大勝ちに引っ張られてプラスに見えているだけ）。年別内訳でも2022〜2025年の大半でリスクオフ・中立がリスクオンを上回る。**サンプルはリスクオフ590件・中立679件と少なめでt値も1.7〜2.0程度に留まるため確定的ではないが、この地合いゲージが「オンで攻め・オフで待て」というツールの想定通りには機能していない可能性が高い**。これは監査P-1〜P-9のバグではなくスコアリング設計上の前提の話であり、本移植の対象外（P-1〜P-9修正の範囲外）として記録するのみに留める。

## 6. 制約事項（必須記載）

1. **P-2（選択バイアス）は今回の設定では実質的に非該当**: 手動ウォッチリストではなく日経225全銘柄（固定・機械的な母集団）を使用したため、「既に上がったから選ばれた」バイアスは大幅に軽減されている。ただし `since` フィールドは全銘柄未設定のままなので、将来ユーザーが独自の値嵩株・テーマ株ウォッチリストに切り替えた場合は、P-2の「検証タブの数字は上限値として読む」という注意が改めて適用される。
2. ストップ高張り付きの厳密な値幅制限テーブルは未実装（ギャップ8%除外で近似）。
3. 検証は「シグナル後N日リターンの統計」であり、ポジション管理・複数保有・資金配分を含むトレードシミュレーションではない。総損益/PF/最大DDは対象外（仕様であり欠陥ではない）。
4. data.js未接続（デモ動作）の場合は検証タブの全数値に投資判断上の意味はない（P-8警告表示済み）。今回はdata.js接続済みで実データ検証だが、上記§4「追加所見」の通り、地合いゲージの想定方向が実データと逆であるなど、スコアリング設計自体の前提には別途疑義が残る。
5. 帰属分析・地合い別内訳は「平均リターン」ベース。地合い別ではサンプル数590〜679と中央値との乖離が大きく、統計的な結論には注意（t値0.5〜2.0程度）。
