## §1 目的と背景（なぜイナゴか・なぜ矛盾しないか）

ユーザーの新方針: **市場トレンドにスムーズに乗る（イナゴ）・資金の流れに敏感になる・
すぐに飛び降りる。** 長期資産形成（NISA）は別枠であり、本ツールは短期の衛星戦略を支援する。

v2/v3で棄却したのは「パターン・オシレーター・トレンド因子の複合」と「指数レベルの遅い退出」。
**「直近上がっている銘柄に乗る（クロスセクショナル・モメンタム）」「出来高急増＝資金流入を
追う」「ポジション単位の速い退出」は未検証**であり、本書はそれを既存データ（5年×223銘柄・
日足OHLCV）で採点する。モメンタムは学術的にも数少ない頑健とされてきたアノマリー系統で、
検証する価値がある。

**評価の哲学は従来どおり**: 候補は手で列挙・基準は事前固定・結果を見てから基準を変えない・
採用ゼロも正当な結果。

## §2 使う部品とデータ（車輪の再発明禁止）

- `evaluate.mjs`: `buildByCode` / `buildDateGrid` / `buildMarketIndex` / `classifyRegimes`。
- CSV読み込みは combo_check.mjs / exit_check.mjs と同じ自前パース（銘柄,日付,OHLCV）。
- **本フェーズは analyze() のリプレイ不要**（終値と出来高だけで全て計算できる）。実行時間は数秒。
- 出来高列（c[6]）は 0/NaN の日がありうる。出来高を使う判定では `volume > 0` の日のみ有効とする。
- 銘柄は共通日付グリッド（`buildDateGrid`）と本数が一致するものだけを使う（seedHistory と同じ
  安全策。ズレのある銘柄はスキップし、使用銘柄数をログに出す）。

### 執行の現実性（本書で最重要の規約）

- **シグナルは当日終値（t）で判定し、執行は翌営業日終値（t+1）とする。** イナゴ系は
  シグナル直後の減衰が速く、ここを甘くすると結果が全て過大評価になる（v3§4と同じ最重要確認点）。
- 参考値として「当日終値執行（執行不能の理論上限）」も併記してよいが、**採否判定は必ず
  t+1執行の数値で行う**。
- **コスト = 片道0.1%（往復0.2%）を事前固定**（大型株のスプレッド＋滑り想定。手数料は無料前提）。
  採否判定は全て**コスト控除後**の数値で行う。

## §3 フェーズG: 検証（momentum_check.mjs 新規・単体実行専用・CI外）

3ブロック構成。出力はテーブル表示＋ `signals/momentum_stats.json`（プルーン正規表現に非該当）。

### G-1. クロスセクショナル・モメンタム（ポートフォリオ方式）

「直近上がっている銘柄を機械的に持ち替え続けたら市場に勝てたか」。

- リバランス: 共通グリッド上の**5営業日ごと**。時点tで全銘柄を過去K日リターン
  `close[t]/close[t-K]-1` で順位付けし、**上位10銘柄**を等ウェイトで保有。執行はt+1終値。
- ルックバック **K=20 と K=60 の2本だけ**（事前固定。グリッド探索禁止）。
- コスト: リバランスごとに入替比率×往復0.2%を控除。
- ウォームアップ: 先頭 K+10 営業日は対象外。
- ベンチマーク: 同期間の等ウェイト指数（`buildMarketIndex`）。
- 計測: コスト控除後の総リターン / 年率超過リターン / maxDD（B&H比）/ 年間売買回転 /
  **ローリング1年窓の超過リターンが正である窓の割合**（頑健性）/
  v3§7の主要下落局面4件それぞれの前後±20営業日の相対成績（**モメンタム・クラッシュ**の確認。
  モメンタムは急落からのV字反発で大負けする既知の弱点がある）。

**採用基準（事前固定）**: コスト控除後の年率超過 ≥ **+3%** かつ ローリング1年超過が正の窓 ≥
**60%** かつ maxDD ≤ B&H の **1.3倍**。

### G-2. 資金流入フォロー（イベント方式・v2と同じ物差し）

「出来高急増＋高値更新に翌日から乗ったら市場に勝てたか」。combo_check.mjs のイベント計測と
同じ構造（同日全銘柄平均をベースラインにした edge）。

- イベント定義は**2種だけ**（事前固定）:
  - `volHigh`: 出来高 ≥ 20日平均出来高×**3** かつ 終値が**60日高値**を更新
  - `volJump`: 出来高 ≥ 20日平均出来高×**3** かつ 当日終値が前日比 **+5%以上**
- 計測: t+1終値エントリーからの 5/10/20日 edge（対市場・方向調整不要、全て買い）。
  **コスト往復0.2%控除後**。勝率・**中央値**（イナゴ系は一部の大勝ちに平均が引っ張られる
  歪んだ分布のはず。平均と中央値の乖離を必ず出す）。レジーム別内訳（classifyRegimes）。
- 参考: 当日終値エントリー版も併記（理論上限のラベル付き）。

**採用基準（事前固定）**: コスト控除後 edge(10日) ≥ **+1.0%**（v2と同じ物差し）かつ
n ≥ **300** かつ edge(20日) > 0。

### G-3. 速い退出の比較（トレードシミュレーション）

「同じ入り方でも、降り方で成績がどう変わるか」。G-2のイベント（`volHigh` を使う。
n<300 なら `volJump`）を入口に固定し、退出規則だけを差し替えて比較する。

- 退出規則は**3種だけ**（事前固定）:
  - `fixed10`: 10営業日後に必ず降りる（ベースライン＝G-2と同じ）
  - `trail5`: 終値が「直近5営業日の最安終値」を割ったら翌日終値で降りる（最長20日）
  - `stop7`: エントリーから **-7%** で翌日終値で降りる（最長20日。利確側は設けない）
- 計測（規則ごと・コスト控除後）: 平均リターン / 中央値 / **ワースト10%の平均**（尻尾の改善が
  「すぐ飛び降りる」の価値）/ 平均保有日数 / 勝率。
- 制約を明記: 日足終値ベースなので場中ストップは再現できない（ギャップで-7%を大きく
  超えて降りる日がある）。結果は概算であり、実運用ではより悪くなりうる。

**判断基準（事前固定）**: `trail5` または `stop7` が `fixed10` に対して
**平均リターンを下げずにワースト10%平均を+2%pt以上改善**したら「退出規則として採用」。
（G-2が見送りの場合、G-3は参考記録のみとし採否判定はしない。）

## §4 フェーズGの実装メモ

- combo_check.mjs / exit_check.mjs の構造（CSV→集計→テーブル＋JSON保存）を踏襲。
- **執行はt+1終値・コスト控除後で判定**（§2の規約。ここが本書の1日先読み対策）。
- G-1のグリッド: 銘柄ごとに `idx.get(date)` で引く（buildByCode）。リバランス時点で
  t-K または t+1 のデータが無い銘柄はその回の順位付けから除外。
- G-2のイベントは銘柄×日の全走査（出来高20日平均はランニング和で）。同一銘柄の連続イベントは
  **10営業日のクールダウン**を置く（同じ急騰に何度も乗る二重計上を防ぐ。事前固定）。
- G-3はG-2で列挙したイベントリストを再利用（再走査しない）。
- 乱数・日時関数は使わない（全て決定的に）。

## §6 制約と正直な注意（結果を読むときに必ず思い出すこと）

- **ユニバースの不一致**: 223銘柄は日経225の大型株。実際のイナゴの主戦場（小型・テーマ株）
  ではない。ここで好結果なら心強いが、悪くても「大型株では効かない」までしか言えない。
  逆に**ここで駄目なのに小型株なら効くはず、と根拠なく飛躍しない**こと。
- **標本は1本の歴史**: この5年は上昇相場が支配的（v2/v3で確認済み）。モメンタムの好成績は
  上昇相場の追い風を含む。ローリング窓の頑健性チェック（G-1）はその緩和策であって解決ではない。
- **執行とコストが命**: イナゴ系の見かけのエッジは執行遅延とコストで消えるのが典型。
  だからt+1執行・コスト控除後で判定する。それでも日足ベースの検証は場中の滑りを含まない。
- **分布の歪み**: 平均がプラスでも中央値がマイナスなら「たまの大勝ちを引くまで負け続ける」
  戦略であり、実運用の心理的持続性は低い。中央値と尻尾を必ず見る。
- AI情報レイヤー（ニュース・決算・テーマ抽出）は本書に含めない。**Gの結果を見てから**
  次の設計（v5）で扱う（価格レイヤーの土台の有無で設計が変わるため）。
